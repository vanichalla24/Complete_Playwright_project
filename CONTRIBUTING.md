# Contributing to QA AI Tool

Thank you for your interest in contributing! 🎉

## How to Contribute

### Reporting Bugs
1. Check if the bug already exists in [Issues](https://github.com/vanichalla24/qa-ai-tool/issues)
2. Create a new issue with:
   - Clear title
   - Steps to reproduce
   - Expected vs actual behavior
   - Browser and OS

### Suggesting Features
Open an issue with the label `enhancement` and describe:
- What problem it solves
- How it should work
- Example inputs and outputs

### Pull Requests
1. Fork the repo
2. Create a branch: `git checkout -b feature/my-feature`
3. Make changes to `index.html`
4. Test in Chrome and Edge
5. Commit: `git commit -m 'Add: my feature description'`
6. Push: `git push origin feature/my-feature`
7. Open a Pull Request

## Code Style
- Keep everything in `index.html` (single-file architecture)
- Add new features to the `FEATS` array and `FORMS` / `PROMPTS` objects
- Follow the existing naming conventions
- Test with a real Gemini API key before submitting

## Feature Ideas Welcome
- New prompt templates
- Additional QA frameworks support
- UI improvements
- Better error messages
- New language support (Java, Python, Ruby)
